# 内容填充与数值设计指南

> 状态：**TARGET**。本文件给出首轮数据生产和调平方法，不宣称任何示例数字是最终历史事实。

## 1. 总原则

1. 先定义关系与制度，再填人物能力。
2. 数值必须解释一种玩家决策，不为“历史味”堆叠无关属性。
3. 同一优势至少有一个成本、依赖或反制。
4. 历史人物不是“名气越大数值越高”；能力评价绑定时期、职位、来源和不确定性。
5. 内容生产与平衡分离：历史事实确定边界，游戏数值在边界内服务可玩性。
6. 不用现代民族/宗教作为天然战斗或生产加成。
7. 所有剧本应支持不同统治路径，不设计唯一正确展开。

## 2. 数据生产顺序

```text
地图节点与路线
→ 人口/农业/市场基线
→ 政体与宗主义务
→ 宫廷/地方职位
→ 历史人物与关系
→ 军事资源与库存
→ 政策/特性
→ 事件与文案
→ 自动跑局调平
```

不要先创建数百武将，再发现地图和制度无法承载他们。

## 3. 规模目标

### 垂直切片

```text
2 个主要政体 + 3 个小型/臣属实体
10 个 settlement
30 个 district
50–80 个人物
4–8 支活动军队
1 个完整农事年
1 条季风显著改变的战役路线
```

### 1.0 内容预算候选

```text
40–60 个可外交政体（并非全部可玩）
250–400 个 settlement
1,200–2,000 个 district
1,500–2,500 个人物
6–10 类地区经济画像
8–12 类政体/职位模板
15–25 项重大政策
60–120 个特性（避免同质百分比）
3 个起始剧本：约 1531、1569、1581
```

1.0 精确规模只在垂直切片与内容工具验证后冻结。

## 4. 人物能力

玩家显示 1–100，内部可以使用整数 0–10000 参与比例计算。建议分布：

| 显示区间 | 含义 | 人群比例目标 |
|---|---|---:|
| 20–39 | 明显不适配或经验不足 | 10–20% |
| 40–59 | 合格地方执行者 | 45–55% |
| 60–74 | 强力人才 | 20–30% |
| 75–89 | 区域杰出 | 5–10% |
| 90–100 | 极罕见且需充分证据 | ≤2% |

约束：

- 人物总评不显示，不允许用总和直接排序所有职位。
- 同一人不应普遍四维全高；卓越人物也有政治、资源和关系限制。
- 年轻人物可用“潜力/经验”影响学习，但不做 RPG 刷级主循环。
- `trait` 应改变规则、偏好或行动集合，不重复做 `+5%` 堆叠。

### 能力赋值证据模板

```text
历史行为：做过什么
资源条件：依赖了什么
反例/失败：何时表现不佳
来源置信度：高/中/低/争议
游戏抽象：哪一项能力/特性表达
平衡约束：为何不应更高或更低
```

## 5. 地区经济画像

每个 District 不逐项手工随意打分。先分画像，再做有限偏移：

- 核心灌溉稻作区；
- 河谷混合农业区；
- 山地/高地交通节点；
- 森林与移动社区区；
- 城市/工匠区；
- 河口/港口贸易区；
- 边境堡垒区；
- 战后受损/人口流失区。

画像定义基准人口、产量、季节性、运输、恢复、税收可实现率。地图团队只修改有依据的偏差。

## 6. 首轮经济单位

选择“一个基础地区一年的可比较预算”为无量纲基准，避免过早伪装真实吨数。

建议：

```text
1 Grain Unit = 1 个标准人口户组约一个月的军民口粮抽象量
1 Cash Unit  = 与标准随从单位一个月维持费可比较的抽象量
1 Labor Day  = 聚合劳役容量，不代表一个现实个体
```

UI 可使用本地化历史术语，但存档和公式使用统一抽象单位。

## 7. 经济基线与约束

目标分布：

- 和平地区正常年份留有 10–25% 可支配余粮；
- 一支大规模农兵远征跨越关键农时，可令来源区下一收获下降 10–35%；
- 常备力量不损农时，但和平年也持续吃掉现金余量；
- 新占地区首年实际税收通常仅为名义值 20–60%，具体取决于战损和安排；
- 直接统治提高长期上限，但短中期行政/驻军成本高；
- 贡赋式臣属收入较低，却减少直接负荷并保留地方动员。

这些是调平起点，不是历史统计。

## 8. 战役数值

### 作战时间

```text
safeCampaignDays = floor(
  (onHandSupply + expectedReliableResupply)
  / max(1, dailyConsumption)
)
```

UI 必须同时显示悲观/中位/乐观区间，不能把不确定采购和臣属履约算成确定库存。

### 有效战力

不以一个总战力决定结果。可用于 AI 粗估：

```text
estimatedForce = manpower
  × readinessFactor
  × moraleFactor
  × supplyFactor
  × cohesionFactor
  × terrainFit
  × intelligenceFactor
```

每个 factor 建议限制在合理区间，如 0.5–1.5，避免百分比乘法爆炸。最终战斗使用分阶段模型。

### 规模优势

两倍兵力不是自动两倍有效战力：路线容量、指挥跨度、粮耗、节点空间和士气降低边际收益。目标是让 1.5–2 倍规模具有显著优势，但良好准备/地形可改变结果；不可让少数神将长期无视十倍兵力。

## 9. 行政与滚雪球控制

```text
load = Σ districtBase
     × communication
     × directness
     × frontierPressure
     × culturalAdministrativeFriction
```

最后一项不是“族群惩罚”，而是语言、制度、地方关系、战争破坏与治理网络的综合摩擦，可被人事、时间和政策改善。

扩张制动：

- 新占控制实现率；
- 驻军与补给；
- 行政负荷；
- 功臣分封导致的地方权力；
- 臣属不可靠；
- 威胁联盟；
- 继承时个人关系重新结算；
- 战争人口/信用损失。

禁止用无解释的全局“扩张惩罚”代替这些系统。

## 10. 关系与义务数值

内部统一用 0–10000 或 -10000–10000；UI 使用描述级别和主要原因。

- `trust` 变化慢，重大履约/背约改变大；
- `affinity` 可受礼物、亲属和性格影响，变化较快；
- `fear` 随展示力量和近期胜负变化，统治者死亡后快速重新评估；
- `interestAlignment` 由结构状态计算，不永久存储为随意数值；
- `outsideOptions` 取决于可投靠的对手与路线。

不要把所有关系压成一个 0–100 好感按钮门槛。

## 11. 特性与政策设计

每项至少回答：

1. 改变什么规则/转换关系？
2. 在什么情境有价值？
3. 代价或暴露面是什么？
4. AI 如何识别？
5. UI 如何解释？
6. 可否与其他项形成无上限乘法？

好例：

```text
河运征发：雨季河运容量提高；地方商贸信任下降，需更高政治容量。
随从整编：常备单位战备恢复加快；月度现金与宫廷派系压力增加。
```

坏例：

```text
勇猛民族：攻击力 +20%
富庶传统：收入 +15%
名将：全属性 +10%
```

## 12. 自动跑局指标

每次平衡批次记录：

- 势力生存曲线；
- 控制/宗主网络集中度；
- 直接吞并与臣属选择比例；
- 战争、撤军、粮尽、雨季困顿率；
- 农业损失与人口恢复；
- 贡赋履约和背约；
- 继承危机频率与结果；
- 政策/特性选择率；
- 人物权力集中度；
- 玩家操作次数预测；
- 决策 trace 的主要阻塞原因。

异常门槛示例：

- 任一政策选择率 >80% 或 <2%；
- AI 因断粮被迫撤退 >25% 的进攻战役；
- 90% 胜者直接吞并，说明臣属方案无价值；
- 一个势力在多数种子 20 年内稳定不可逆滚雪球；
- 同一人物组合成为所有职位的最优解。

## 13. 调平流程

```text
提出玩家体验假设
→ 指定可观察指标
→ 建立最小场景/批量种子
→ 只改少量参数
→ 运行并比较分布
→ 人工检查典型回放
→ 记录结论和副作用
```

不要边跑边随意改几十个数，也不要只根据单局故事调平。

## 14. 内容审查清单

- 是否有来源/置信度？
- 是否把现代身份倒投到 16 世纪？
- 是否把文化写成固定属性？
- 是否把暴力、奴役或迁徙变成纯收益？
- 是否存在同类但更弱、永远不选的内容？
- 是否对 AI 和 UI 有解释？
- 是否引入组合爆炸或不可测试例外？
