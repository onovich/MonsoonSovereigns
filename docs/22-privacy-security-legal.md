# 隐私、安全、许可与法律基线

## 1. Offline-first

1.0 是单机游戏，核心模拟、存档和内容不需要账户或常驻后端。默认不收集遥测，不以“未来可能联网”为由预埋宽权限服务。

## 2. 可选诊断

若后续加入崩溃/性能诊断：

- 明确 opt-in 或符合发行地要求的同意；
- 最小化数据；
- 不上传存档、人物命名、文件路径或线程日志全文；
- 提供查看/删除/关闭；
- 在隐私说明列出处理者、保留期和用途；
- 通过 ADR 和安全审查。

## 3. Electron/Web 安全

- Renderer 无 Node；preload 白名单；IPC schema；CSP；可信 origin；
- 存档、内容包和 Mod 均是不可信输入；
- Web 部署启用 HTTPS、正确 MIME、缓存版本化、无远程 eval；
- 外链必须确认并经 allowlist；
- 不从 CDN 动态加载未锁定代码。

## 4. 开源许可

每项依赖记录 SPDX、版本、版权声明和分发义务。发布生成：

- third-party notices；
- SBOM；
- source offer/归属（若许可证需要）；
- 字体、音频、图像、地图数据的独立许可证清单。

“网上可下载”不等于可商用。博物馆图片、历史地图扫描和现代转写可能各有权利。

## 5. 历史人物与文化资产

多数 16 世纪事实本身不受版权保护，但：

- 现代书籍的表达、译文、地图、插图和照片受保护；
- 不复制原作游戏 UI、文案、武将图、数据表或专有资产；
- 对现实宗教/文化符号审慎，检查发行地区法律和平台规范；
- 品牌名《季风诸王》仅是工作名，商标检索后再冻结。

## 6. AI 产出治理

- 每项提交仍由项目负责，不以“模型生成”免除许可、正确性或文化责任；
- 不把受版权保护的大段文本/图像直接塞入仓库；
- 研究笔记以摘要、引用限额和出处为主；
- 不提交模型凭证、会话秘密或用户私有数据；
- 正式内容保留人类/高能力 reviewer 的可追溯批准。

## 7. 存档和用户文件

- 写入仅限应用数据目录或用户明确选择路径；
- 防路径穿越、符号链接意外覆盖和压缩炸弹；
- 原子写和备份；
- 卸载默认不静默删除用户存档；
- 导入前校验大小、格式、版本和 checksum。
