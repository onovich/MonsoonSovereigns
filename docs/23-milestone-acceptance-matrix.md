# 里程碑验收矩阵

本表供协调线程决定是否允许进入下一阶段。

| Gate | 产品/设计 | 模拟/数据 | 客户端/平台 | QA/研究 |
|---|---|---|---|---|
| M0 | 范围和非目标可见 | 跨 Node/Worker hello hash | Web/Electron 空壳 | CI、Skill 路由演练 |
| M1 | Command 语义明确 | 10 年 deterministic、save v1 | 最小 read model | invariant/golden/协议测试 |
| M2 | 季节和经济形成选择 | 农事、人口、路网守恒 | 地图、列表、压力 fixture | H-001/H-002 初验、地图来源政策 |
| M3 | 三种统治方式有差异 | 政体、义务、人物、继承最小 | 任命、批量、理由 UI | H-003/H-005 初验、历史 claim pipeline |
| M4 | 战役闭环成立 | 集结、补给、情报、战斗/攻城 | 战役规划器、战报 | 撤军/断粮/季风场景，AI trace |
| M5 | 2–4 小时纵切可重复 | AI 多年运行无救援 | Web/Windows 可分发 | 玩家研究、性能门、手动会战决定 |
| M6 | 从开局到胜利系统完整 | 全系统 AI、三剧本管线 | 教学/百科/可访问核心 | Alpha 无 P0/P1、顾问介入 |
| M7 | 内容锁定 | 大规模平衡/迁移 | 平台/分辨率/本地化矩阵 | Beta 文化审查、人工通关 |
| M8 | 无大系统变化 | release hash/存档稳定 | 安装/更新/回滚 | 安全、SBOM、发布检查 |

## Gate 规则

- 任一列有 `BLOCKED` 即不能升阶段。
- “有功能”不等于通过；必须有验收证据和 reviewer。
- 失败可降低范围、修改 TARGET 或回到前阶段，但冻结决定需 ADR/人类批准。
- 每个 Gate 生成一份 `GATE-<MILESTONE>.md`，列测试、数据、已知风险和明确结论。
