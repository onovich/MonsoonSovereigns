## R Gate 规则

- 任一列有 `BLOCKED` 即不能升阶段。
- “有功能”不等于通过；必须有验收证据、运行时证据、负向证据和 reviewer。
- DOM 元素、`data-*` 属性、fixture 文本、无头 Runner、Storybook 点击或构建退出码不能单独支持产品 Gate。
- 失败可降低范围、修改目标或回到最近满足定义的产品阶段；冻结决定仍需 ADR/人类批准。
- R0-R6 Gate 生成对应 `GATE-R*.md` 或任务内等价验收记录时，必须列测试、数据、已知风险和明确结论。
- R7 和 R8 必须由产品所有者 Human Gate 明确通过；核心流程缺失、真实运行时未闭环或地图方向未接受时，不得用 `PASS_WITH_LIMITS` 绕过。
