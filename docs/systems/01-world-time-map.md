# 系统规格 01：世界、时间与地图

## 1. 设计目标

地图表达的是交通、人口集中和权力可达性，而非现代行政边界。玩家应感受到河谷、山口、旱季道路和港口比空白面积更重要。

## 2. 空间层级

```text
Region（大区，仅用于统计/叙事）
└─ District（基础人口与控制单位）
   ├─ Settlement（城、镇、寺院、港口、堡垒）
   └─ RouteNode / RouteEdge（道路、河运、山道、沿岸航线）
```

- `District` 有且仅有一个实际控制者。
- 宗主、宣称、宗教联系、贸易依赖不改变实际控制字段。
- 边界可模糊地表现，但模拟必须有唯一归属以保持可验证性。
- 城不是唯一价值容器；人口、稻作区、渡口、矿产和商路可位于不同 District。

## 3. 路网

每条边至少包含：

```text
length
mode: road | river | mountain | coastal
seasonalPassability
baseCapacity
terrainCost
controlRisk
knownStateByFaction
```

同一图用于行军、补给、贸易、情报、行政通信和逃亡。不同用途可有不同 cost function，不复制多张彼此漂移的图。

## 4. 时间

权威最小步长为一天。日历可采用历史月份显示，但模拟不依赖现实系统时钟。

- 每日：行军、补给、围城、疾病、侦察、局部建设进度。
- 每月：财政、薪俸、贡赋检查、忠诚、提案、战略 AI。
- 农事阶段：整地/播种、维护、收获；不同农业区的日期可配置。
- 年度：长期人口、继承年龄、威望衰减、制度评估。
- 事件触发：会战、投降、死亡、叛乱、背约。

## 5. 季节与季风

季风是多个参数的共同来源，不是统一减速 debuff：

- 道路可通行度；
- 河流水位与运输能力；
- 海上航行窗口；
- 农业劳力需求；
- 疾病/牲畜损耗；
- 围城工事和火器可靠性；
- 地方粮食采购能力。

首版使用区域化季节曲线，不尝试逐日气象。天气事件仅做低频修正。

## 6. 雾与认知地图

每个势力维护 `FactionKnowledgeState`：

- 最后观察日；
- 兵力/粮食估计范围；
- 来源与可信度；
- 路线是否已知；
- 传闻与确认信息分离。

UI 不向玩家泄漏真实世界状态；AI 也只读取自身知识。

## 7. 不变量

- 每个 District 恰有一个控制者或显式 `Uncontrolled`。
- 路线端点有效，地图关键区可达性通过内容构建验证。
- 单位每天最多提交一次权威位置转移。
- 季节函数对同一 day/region 确定。
- 地图视觉坐标不参与权威距离判定。

## 8. 垂直切片

- 30 District、10 Settlement、约 60 路线边。
- 至少一条雨季变差的道路和一条雨季改善的河运路线。
- UI 可展示预计行程、容量、季节变化和情报置信度。
