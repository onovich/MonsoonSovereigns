# 美术、音频与本地化方向

> 状态：**TARGET / RESEARCH REQUIRED**。尚未冻结具体画风。

## 1. 表现目标

- 信息优先，但不能做成无质感后台表格；
- 让河流、季风、宫廷和权力网络成为视觉主题；
- 展示区域差异，不把东南亚做成单一“热带异域”；
- 大地图可读、低端设备可运行、Web 首次加载可控。

## 2. 推荐视觉路线

候选：**2D/2.5D 地图 + 平面/绘画人物肖像 + 有限骨骼/Shader 动效**。

避免首版：

- 全 3D 写实城市；
- 大规模单兵模型；
- 每个文化完全独立高成本资产集；
- 把历史图像直接当无版权背景拼贴。

### 地图层

- 地形以河谷、山口、稻作、森林、港口和季节水位为重点；
- 政治边界为柔性控制/影响视觉，不画现代国界硬线；
- 雨季/旱季通过路线、河面、植被和氛围变化，而非整屏滤镜；
- 地图模式与基础美术分层，保证色觉可读。

### 人物层

- 肖像区分场合、阶层、年龄和地区；
- 不凭现代节庆服装或影视戏服直接还原；
- 史料不足可采用克制、统一的“历史启发”语言并标注；
- 女性、僧侣、商人、使者和非军事角色必须有完整视觉地位。

## 3. UI 视觉

- DOM 原生文字和表格，不把 UI 烘焙进低分辨率 Canvas；
- 装饰不遮挡数字、原因和预测；
- 历史纹样仅在确认语境后使用；
- 神圣符号不作随意按钮或货币图标；
- 重要状态同时使用形状、文字和颜色。

## 4. 音频

- 音乐定位为“受历史区域材料启发的原创”，不宣称精确复原；
- 研究乐器、音阶、场合和现代传统变迁；
- 不把不同地区音乐混成泛亚洲音色库；
- 战争声音强调距离、环境、命令和士气，不追求持续轰炸；
- UI 音效提供层级和可关闭选项。

## 5. 本地化

首发语言优先级由商业阶段决定，但系统必须从 M0 支持：

- Unicode 完整；
- ICU/具名变量；
- 复数/语序不靠字符串拼接；
- 名称、称号和别名独立；
- CJK 字体按语言/剧本子集加载；
- 地图标签和 DOM 字体策略分离。

历史称号的显示结构：

```text
玩家易懂称呼
原文/转写（百科或 tooltip）
任职时间与别名
来源/不确定性（百科）
```

## 6. 资产生产门禁

每类正式资产需：

1. reference board 带来源、年代、地域；
2. historical_researcher 审核；
3. 文化敏感性检查；
4. 性能/尺寸预算；
5. 许可证与 provenance；
6. 引擎中实际分辨率/色觉/缩放验证。

AI 生成图只可用于内部构图探索，未经版权、文化和一致性审查不得直接进入正式产品。
