# 执行交接书

本仓库自此作为《季风诸王》的唯一项目事实源。接手方被授权在冻结边界内独立完成研究、设计、实现、测试和集成，不应依赖此前会话上下文，也不应等待原规划者继续补充。

## 1. 你正在做什么

一款受《信长之野望》“土地—家臣—动员—战争—赏罚”组织循环启发、但不复制其题材和按钮的单机大战略游戏。舞台为约 1531—1600 年东南亚大陆。玩家通过宫廷、职位、地方领主、臣属王、贡赋、人质、婚姻、季风窗口和补给维持霸权。

产品的判断题不是“兵种够不够多”，而是：

> 失去一名可信且能干的地方统治者，是否会迫使玩家改变整个战略？

## 2. 已完成的规划

- 产品支柱、题材与平台已经冻结。
- GDD、程序设计、技术栈、架构、系统规格、数值指南、历史政策、roadmap 和风险均已提供。
- Codex 自定义角色、模型分工、线程路由 Skill、任务/交接 schema 和 M0/M1 backlog 已提供。
- 尚无真实产品代码、地图数据库、美术或已验证玩法原型。

## 3. 接手方的首要职责

不是继续扩写愿景，而是建立可重复的工程基线，并逐里程碑证明假设：

```text
M0 工具链与协作
→ M1 确定性模拟
→ M2 世界/经济/人口
→ M3 人物/臣属/治理
→ M4 战役/补给/攻城
→ M5 可玩垂直切片
```

任何 milestone 未通过退出条件，不得靠增加内容掩盖。

## 4. 权限边界

接手方可以：

- 在 TARGET/HYPOTHESIS 范围内通过测试调整规格；
- 创建 ADR；
- 改善代码、工具、测试和内容管线；
- 发现史料不足时标记并重新研究；
- 拒绝不成立的实现路径。

接手方不能在普通任务中：

- 改题材、核心支柱、Web-first、TypeScript/Worker/React/Pixi/Electron 基线；
- 将游戏改成现代国族涂色、3D RTS、多人服务或抽卡 RPG；
- 绕过确定性、玩家/AI 同命令、独立审查和历史来源要求；
- 用“AI 生成内容”替代史料和文化审查。

必要变更按 ADR、原型证据和人类产品所有者批准处理。

## 5. 启动指令

使用 `bootstrap/CODEX_FIRST_PROMPT.txt`，显式调用 `$monsoon-studio-orchestrator`。先完成 `FOUNDATION-001`，不提前实现玩法。

## 6. 交付标准

执行者的工作只有在以下全部成立时才完成：

- 代码/内容可在干净环境复现；
- 测试和验收证据真实；
- reviewer 独立接受；
- 规则、schema、文档同步；
- 历史内容可追溯；
- 未完成和不确定项诚实持久化；
- 下一线程通过 handoff 的 `ROUTE_TO` 得到明确输入。
